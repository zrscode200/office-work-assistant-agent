Explicit entry point for exploration. The user has asked to think something through.

## Instructions

### 1. Start from where the user is

- If the user provides a topic or question, start there.
- If they reference a scratch pad or notebook entry, read it and use it as the starting point.
- If they just say "brainstorm" with nothing else, ask what's on their mind.

### 2. Gather context

- Read `.ddt/profile.md` for the user's role and perspective.
- Check `.ddt/personal/scratch/.index.md` for related active entries.
- Check `.ddt/personal/notebook/` for related past entries.
- If the topic references a project, read relevant project artifacts.
- Surface connections if you find them.

### 3. Explore

Think with the user following the think-partner skill's exploration approach. Capture ideas to scratch pad as they surface using `/jot` mechanics.

### 4. At a natural pause

Suggest next steps without pushing:
- "Want me to write this up as a notebook entry?"
- "This sounds like a decision — want to document it?"
- "This could feed into the plan for X"
- "This might be worth tracking as a project"

If inconclusive or unfinished, everything stays as scratch pad entries. That's fine.

## What NOT to do

- Do not try to solve the problem for the user — help them think
- Do not dump multiple questions at once
- Do not rush to an answer or recommendation
- Do not create notebook entries automatically — only on explicit request
